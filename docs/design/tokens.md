# Design Tokens — Legacy vs Rebuild Audit

> **Source of truth:** this document compares legacy `nawhas.com` tokens (extracted code-first from `github.com/nawhas/nawhas@master` via `gh api`) with the current rebuild's tokens (from `apps/web/app/globals.css` and related files). Every legacy value carries a provenance citation `nawhas/nawhas:<path>#Lx-y`. Values that could not be extracted code-first are marked `deferred: visual verify` — see `README.md` for the agenda.
>
> **Action column:**
> - `keep` — rebuild value already matches legacy intent
> - `replace with <value>` — rebuild must adopt the legacy value in Phase 2.2
> - `add` — legacy has this, rebuild does not, must be added in 2.2
> - `drop` — rebuild has this, legacy doesn't, and 2.2 should remove (rare; only if it actively conflicts)

## Typography

Legacy loads five Google Fonts at `nawhas/nawhas:nuxt/nuxt.config.js#L210-219` (Bellefair, Roboto, Roboto Slab, Roboto Mono, Material Icons + Material Icons Outlined), but only Roboto is wired to Vuetify's SCSS font vars — Vuetify 2's `$body-font-family` and `$heading-font-family` are **not** overridden in `nuxt/assets/variables.scss`. Bellefair, Roboto Slab, and Roboto Mono are applied exclusively in component/page `<style>` blocks (see citations per row). The heading type-scale (h1–h6 sizes/weights/line-heights/letter-spacing) and body/caption/subtitle/overline/button roles all defer to Vuetify 2's upstream `$headings` map and legacy-named type tokens; the only local scale tweak is `.section__title` (`nawhas/nawhas:nuxt/assets/app.scss#L6-12`), which pins `font-size: 1.4rem` while reusing the h5 weight/line-height/letter-spacing/family via `map-deep-get($headings, 'h5', ...)`. The rebuild declares three font-family CSS variables in `globals.css` but ships **no explicit type scale** — heading sizes come from Tailwind's utility defaults, not from design tokens.

## Spacing

Legacy ships **no local spacing-scale override** — `nawhas/nawhas:nuxt/assets/variables.scss` is 10 LOC and only overrides `$container-max-widths` (lg/xl → 1200px). The de-facto scale is Vuetify 2's `$spacer: 4px` base plus its `ma-0`..`ma-16` / `pa-0`..`pa-16` helper ladder (0, 4, 8, 12, 16, 20, 24, 28, 32, 36, 40, 44, 48, 52, 56, 60, 64 px). Page-level rhythm comes from a handful of ad-hoc literals in `nawhas/nawhas:nuxt/assets/app.scss` (section padding, section-title margin, responsive page gutter, persistent-player reserve). The rebuild uses **Tailwind 4's default 4px base spacing** (same primitive as Vuetify) but declares an explicit breakpoint ladder in `apps/web/app/globals.css` `@theme` that diverges from Vuetify 2's `$display-breakpoints` at every named step. Container widths and page gutters are component-scoped in the rebuild — encoded inside `apps/web/src/components/layout/container.tsx` rather than as design tokens.
